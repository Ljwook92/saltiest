1 SQL 타입

  • SQLite : 작은 규모의 프로젝트, 테스트, 프로토타입 또는 모바일 애플리케이션에 적합
  • MySQL : 중소규모에서 대규모 프로젝트, 웹 애플리케이션 또는 비즈니스 응용 프로그램에 적합
  • PostgreSQL : 대규모 및 고급 프로젝트, 데이터 일관성이 중요한 응용 프로그램에 적합

2 SQL 장단점

타입 장점 단점
SQLite
  • 가볍고 편리
  • 오픈소스
  • 동시성 제한
  • 확장성 제한
MySQL
  • 성능 및 확장성
  • 커뮤니티
  • 다양한 플렛폼
  • 보안상 취약점 발생 가능
PostgreSQL
  • 고급 데이터베이스 기능
  • 확장성 및 성능
  • 오픈소스 및 활발한 커뮤니티
  • 일부 초기 설정 필요

SQLit은 파일 기반 데이터베이스로서 별도의 서버 프로세스가 필요하지 않음, 단일 파일에 모든 데이터를 저장. 이로 인해 배포 및 사용이 간편. 또한 오픈소스이기 때문에 대부분의 플렛폼에서 지원. 그러나 동시에 여러 사용자가 데이터를 수정하는 경우 성능이 감소할 수 있고, 대용량 및 고성능 데이터베이스로는 부적합함.

MySQL은 대규모 데이터베이스 및 요청을 처리할 수 있는 높은 성능과 확장성을 제공, 큰 커뮤니티와 풍부한 문서화로 사용자들 간 지원이 용이, 또한 다양한 운영 체제 및 플렛폼에서 지원. 그러나 초기 설정에서는 보안상 취약점이 발생할 수 있기 때문에 주의.

PostgreSQL은 ACID 준수, Foreign Key, Trigger, View, Save procedure등 고급 데이터베이스 기능을 지원. 대용량 데이터 처리 및 복잡한 Query 에 적합한 뛰어난 성능과 확장성, 강력한 커뮤니티를 갖고 있기 때문에 지속적인 개발과 지원이 가능. 그러나 초기 설정이 MySQL에 비해 더 많은 작업을 필요.

4 SQL Packages

각 SQL 타입에 맞는 R패키지

4.1 SQLite :

주로 개인 컴퓨터의 데이터베이스로 사용됨. 저장 경로는 getwd()를 통해 확인 (R 기본 경로)

install.packages("RSQLite") 
install.packages("DBI")
library(RSQLite) 
library(DBI)

con <- dbConnect(RSQLite::SQLite(),           
1                 dbname = "path/your_database.db")
1
dbConnect 함수를 통해 데이터베이스와의 connection을 생성
2
dbname : 데이터베이스 이름 (변경 가능)

NOTE: 업로드 될 모든 파일들은 “your_database.db” 파일안에 각각 설정된 데이터명으로 저장

4.2 MySQL :

기업 또는 학교에서 만든 데이터베이스를 접근 할 때 사용

install.packages("RMySQL")
install.packages("DBI")
library(RMySQL)
library(DBI)

con <- dbConnect(RMySQL::MySQL(), 
                 dbname = "your_database",
                 host = "your_host",
3                 user = "your_user",
                 password = "your_password")
3
host,user,password 는 개인 데이터베이스가 아닌 일정 규모 이상의 그룹 데이터베이스에 접근하기 위해 필요

4.3 PostgreSQL :

MySQL과 유사

install.packages("RPostgreSQL")
install.packages("DBI")
library(RPostgreSQL)
library(DBI)

con <- dbConnect(RPostgreSQL::PostgreSQL(), 
                 dbname = "your_database",
                 host = "your_host", 
                 user = "your_user", 
                 password = "your_password")

실습으로 host, user, password가 필요없이 간단히 사용 가능한 SQLite 패키지를 사용

5.4 간단한 Query 사용하기

SQL에서도 R처럼 Query 를 사용하여 기본적인 작업이 가능함. 예를들어 위에서 실행한 head(), group_by(), 와 summarize() 역시 SQL에서 사용 가능

dbGetQuery(con, "SELECT * FROM iris_data LIMIT 5;")
  Sepal_Length Sepal_Width Petal_Length Petal_Width Species
1          5.1         3.5          1.4         0.2  setosa
2          4.9         3.0          1.4         0.2  setosa
3          4.7         3.2          1.3         0.2  setosa
4          4.6         3.1          1.5         0.2  setosa
5          5.0         3.6          1.4         0.2  setosa
dbGetQuery(con, 
           "SELECT Species,
           AVG(Sepal_Length) AS Mean_Length,
           AVG(Sepal_Width) AS Mean_Width
           FROM iris_data
           GROUP BY Species;")
     Species Mean_Length Mean_Width
1     setosa       5.006      3.428
2 versicolor       5.936      2.770
3  virginica       6.588      2.974

사용을 다 한 후에는 데이터베이스 연결을 해제

dbDisconnect(con)
